## 介绍

图形组是 TDS 程序组下面建立的新组别，和客户端方向并行。

建立图形组的目的是探索有趣的游戏世界的知识，游戏最终呈现给玩家的是另一个奇妙的世界，而我们的目的是探究支撑这个世界运转的原理，并亲手把它们实现出来——这里指的大部分是游戏图形。当然，除了复刻现实世界的视觉真实感之外，还可以像印象派画家一样创造出一个风格化的世界。

创始人布朗尼蛋糕以及常驻组内的校外朋友们具有非常浓厚的图形技术兴趣，并且是非功利性质的，不会采取任何大环境导向的叙事——在这里热爱说了算，会尽力打造一个没有压力、轻松愉快的技术氛围。

## 目标

成为合格的游戏图形开发者。最低标准水平是 GAMES202 + 104 (渲染部分) 以上，并有能力自行探索高级图形技术。

在图形学基础上超越本校图形图像硕士研究生水平，在工程能力上达到可以根据自己的想法做出作品集的水平。

## 组内学习方案

当然不是一上来就要学 games 课程什么的，其实是等大家跟着程序大组把 OOP 熟悉一遍之后（训练出一点编码的感觉），再上点强度学习正经的图形知识。

### 第零阶段

在第零阶段，主要是让大家去看一些高质量的图形学科普视频，并且结合里面的一些技术名词分析自己喜欢玩的游戏利用了什么技术，从顶层（效果层面）先建立起一个对游戏图形技术的感觉。这大概只用花两周的时间。

**通过本阶段的学习，你将可以：**
1. 分析基本的游戏图形技术。
2. 了解哪些图形技术会很吃资源，并能更聪明地调节游戏画质设置。
3. 对游戏图形的发展有基本的感觉。

### 第一阶段

过了第零阶段，就需要学习一些基本的数学知识，它们将成为理解游戏图形的基础：

- 常用数学
  - 线性代数
  - 微积分
  
放心，基本要求就只是先让大家能看懂符号表达式的含义，而不是花很多时间讲令人困惑的细节或是做很多题。后者的职能已经由本校的《高等数学》、《线性代数》课程所分担 ~~虽然学校讲得依托~~。

**通过本阶段的学习，你将可以：**
1. 大脑升级，，，
2. 能看懂技术文章里的数学公式想表达什么
3. 高数线代挂科的概率降低很多

### 第二阶段

这个阶段将正式开始学习计算机图形学。

首先，我们不会遵循传统路径——也就是绝大多数计算机图形学课程的先光栅化再光追，相反地，我们是先光追再光栅化。这其中有多方面的考量，会在详细的图形文档里说明。
